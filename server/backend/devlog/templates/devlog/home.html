Project FAQ

Why Django Rest Framework?

Django Rest Framework (DRF) offers an easy way to build secure and scalable APIs. It integrates seamlessly with Django's ORM and provides built-in features like authentication and permissions. Compared to Flask or FastAPI, DRF is more feature-rich and well-suited for large projects that need more than just a simple API.

Why React.js?

React.js allows for fast development with its component-based architecture and unidirectional data flow. It‚Äôs more flexible and easier to set up compared to Angular or Vue.js, with a vast ecosystem that supports scalability, making it a solid choice for modern web apps.

Why PostgreSQL?

PostgreSQL excels in handling relational data with strong support for ACID compliance, indexing, and complex queries. Unlike MySQL, it‚Äôs more reliable for large-scale applications that require robustness and performance, making it a great fit for our backend needs.

Why Simple JWT?

Simple JWT is lightweight and easy to implement for stateless authentication. It provides a scalable solution without the need for server-side sessions, making it more efficient than traditional methods like OAuth2, especially for modern frontend-backend communication.

Why GitHub Actions?

GitHub Actions allows for seamless CI/CD workflows directly within GitHub, offering easy integration and automation. Unlike Jenkins, it‚Äôs simpler to set up and maintains better integration with the repository, streamlining the development process with testing and deployment automation.

Why Railway and Netlify?

Railway offers easy deployment and scaling for backend services, with a free tier and PostgreSQL support. Netlify is perfect for deploying frontend React apps with automatic CI/CD, making them a cost-effective, developer-friendly choice compared to AWS or Heroku.

‚úÖ Solved with Simple JWT, custom permission classes, and a unified token strategy.

üß† Designing Flexible Course & Content Models

Supporting diverse content types while preserving structure and sequence took multiple iterations.

‚úÖ Solved with ordered content models, content-type validation, and modular architecture.

‚úÖ Solved by configuring GitHub Actions with PostgreSQL service, Django testing, and SonarCloud analysis.

üîÄ Migration Conflict Management

‚úÖ Solved by sequencing migrations carefully, using `--merge`, and improving communication during DB schema changes.

‚úÖ Solved with environment syncing, PostgreSQL backup scripts, and CORS/debug strategies per environment.
